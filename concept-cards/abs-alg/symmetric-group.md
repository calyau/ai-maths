# Quick Definition
The symmetric group $S_n$ is the group of all permutations (bijections) of $\{1, 2, \ldots, n\}$ under function composition. It has order $n!$ and is non-abelian for $n \geq 3$.

# Core Definition
Let $\Omega$ be any nonempty set. The *symmetric group on $\Omega$*, denoted $S_\Omega$, is the group of all bijections from $\Omega$ to itself under function composition. When $\Omega = \{1, 2, \ldots, n\}$, this is denoted $S_n$, the *symmetric group of degree n*. The order $|S_n| = n!$. The structure of $S_\Omega$ depends only on $|\Omega|$, not on the particular elements (Dummit & Foote, pp. 29-31).

# Prerequisites
- **Group** — $S_n$ is a group
- **Bijection** — elements are bijections (permutations)

# Key Properties
1. $|S_n| = n!$
2. $S_n$ is non-abelian for $n \geq 3$
3. Every permutation has a unique cycle decomposition (up to order and cyclic permutation within cycles)
4. The order of a permutation equals the l.c.m. of its cycle lengths
5. Disjoint cycles commute
6. $S_n$ is generated by transpositions

# Construction / Recognition
## To Construct/Create:
1. List all bijections from $\{1, \ldots, n\}$ to itself
2. Compose as functions (right to left)

## To Identify/Recognize:
1. Any group of permutations of a finite set of n elements embeds in $S_n$

# Context & Application
$S_n$ is fundamental to group theory. Every finite group is isomorphic to a subgroup of some $S_n$ (Cayley's Theorem). Symmetric groups are key in Galois theory and combinatorics.

# Examples
**Example 1** (p. 31): The 6 elements of $S_3$: $1, (12), (13), (23), (123), (132)$.
**Example 2** (p. 31): $(12) \circ (13) = (132)$ and $(13) \circ (12) = (123)$, showing non-commutativity.

# Common Errors
- **Error**: Reading composition left-to-right instead of right-to-left. **Correction**: In $\sigma \circ \tau$, apply $\tau$ first, then $\sigma$.

# Common Confusions
- **Confusion**: Confusing elements of $S_n$ (permutations) with elements of $\{1,\ldots,n\}$ (the set being permuted). **Clarification**: Elements of $S_n$ are *functions*, not integers.
