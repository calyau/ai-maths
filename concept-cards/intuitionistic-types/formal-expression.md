# Quick Definition

A formal expression is a syntactic entity generated by rules mirroring the type and term formation rules, but without any requirement of well-typedness. Expressions include all types and terms but also meaningless combinations.

# Core Definition

Martin-Lof defines formal expressions (2.4.1) by a simultaneous induction that closely parallels the rules for types and terms. The key difference is that no typing constraints are imposed: any expression may be combined with any other according to the syntactic shape of the constructors. The class of expressions includes:

- Type constants applied to expressions: P(a_1, ..., a_n)
- Pi and Sigma forms: (Pi x in A)B[x], (Sigma x in A)B[x]
- Sum: A + B
- Base types: N_0, N_1, ..., N, V
- Variables and object constants
- Lambda abstraction: (lambda x)b[x]
- Application: b(a)
- Pairing: (a, b)
- Sigma elimination: E(c, (lambda x)(lambda y)d[x,y])
- Injections: i(a), j(b)
- Definition by cases: D(c, (lambda x)d[x], (lambda y)e[y])
- Finite type elements: 1, ..., n
- Finite type elimination: R_n(c, c_1, ..., c_n)
- Zero and successor: 0, s(a)
- Recursion: R(c, d, (lambda x)(lambda y)e[x,y])

# Prerequisites

No prerequisites beyond familiarity with the term and type formation rules of the theory (sections 2.2-2.3).

# Key Properties

1. Every type is an expression, and every term is an expression, but not every expression is a well-typed term or a type.
2. The class of expressions is defined purely syntactically -- no type-checking is involved.
3. The broader class is introduced specifically so that the Church-Rosser property can be proved without reference to typing.
4. After Church-Rosser is established for expressions, the result that types and terms are closed under reduction (Theorem 2.4.7) brings the property back to the well-typed world.

## To Identify/Recognize:
An expression is any syntactic entity built by the rules of 2.4.1. It need not be a well-typed term or a well-formed type.

# Context & Application

The notion of formal expression is a technical device. Martin-Lof introduces it because proving the Church-Rosser property requires reasoning about arbitrary syntactic forms, including intermediate results of contraction that have not yet been verified to be well-typed. Once Church-Rosser is established for expressions, the closure theorems (2.4.7, 2.4.8) show that reduction never leaves the world of well-typed terms and types.

# Examples

The expression (lambda x)x(x) is a formal expression (self-application), even though it is not a well-typed term in the theory of types.

Any well-typed term such as (lambda x)b[x](a) is also an expression, and its contractum b[a] is likewise an expression.

# Common Errors

- **Error**: Assuming that all expressions are meaningful (well-typed).
  **Correction**: Expressions are purely syntactic. Well-typedness is a separate property established by the type and term formation rules.

# Common Confusions

- **Confusion**: Why introduce meaningless expressions at all?
  **Clarification**: The Church-Rosser property must be proved for a class closed under arbitrary subexpression replacement. The well-typed terms alone do not form such a class, so the broader class of expressions is needed.
