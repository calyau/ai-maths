# Quick Definition

The type formation rules (Section 2.2) specify how all types of the theory are generated, by simultaneous induction with the term formation rules.

# Core Definition

Martin-Lof writes in Section 2.1: "By a simultaneous induction, we shall generate certain symbolic expressions called *types* and, for every type, certain other expressions called the *terms* of that type."

Section 2.2 gives the following type formation rules:

- **2.2.1** (Type constants): If P is an n-ary type constant with arguments of types A_1,...,A_n[x_1,...,x_{n-1}] and a_1,...,a_n are terms of those types, then P(a_1,...,a_n) is a type.
- **2.2.2** (Pi): If x is a variable of type A and B[x] is a type, then (Pi x in A)B[x] is a type. When B does not contain x free, this is abbreviated A -> B.
- **2.2.3** (Sigma): If x is a variable of type A and B[x] is a type, then (Sigma x in A)B[x] is a type. When B does not contain x free, this is abbreviated A x B.
- **2.2.4** (Plus): If A and B are types, then A + B is a type.
- **2.2.5** (Universe): V is a type.
- **2.2.6** (Small types): A term of type V is a small type. A type is large if and only if it contains an occurrence of V or a type constant P.

# Prerequisites

- type: The informal notion of type from Chapter 1.
- variable: Variables with their types appear in rules 2.2.2 and 2.2.3.
- dependence: The dependent type formers (Pi, Sigma) require understanding dependence.

# Key Properties

1. Types and terms are generated by simultaneous induction -- rule 2.2.6 links the two by declaring that terms of type V are themselves (small) types.
2. The Pi and Sigma type formers are the dependent generalizations of function types and product types.
3. The non-dependent special cases are A -> B (function type) and A x B (Cartesian product).
4. A type is small if and only if it is a term of V; it is large otherwise.
5. It can be mechanically decided whether a type is small or large: a type is large if and only if it contains an occurrence of V or a type constant P.

# Construction / Recognition

## To Form a Type:
1. Start with type constants applied to terms (2.2.1), the universe V (2.2.5), or small types from V (2.2.6).
2. Build compound types using Pi (2.2.2), Sigma (2.2.3), or + (2.2.4).

## To Classify:
- A type is small if it is a term of type V.
- A type is large if it contains V or a type constant.

# Context & Application

The type formation rules formalize the informal explanations of Chapter 1. They correspond to logical connective formation in first order logic: Pi corresponds to universal quantification, Sigma to existential quantification, + to disjunction, and type constants to predicate constants. The simultaneous generation of types and terms (via rule 2.2.6) is the formal expression of the universe construction.

# Examples

- If A is a type and B[x] is a type for x of type A, then (Pi x in A)B[x] is a type.
- If A and B are types with no dependency, then A -> B, A x B, and A + B are types.
- N is a term of type V (by V-introduction, rule 2.3.13), so N is a small type.
- V itself is a large type.

# Common Errors

- **Error**: Forgetting that type and term generation are simultaneous.
  **Correction**: Rule 2.2.6 makes terms of type V into types, so the generation of types depends on the generation of terms and vice versa.

# Common Confusions

- **Confusion**: Treating A -> B and (Pi x in A)B as different type formers.
  **Clarification**: A -> B is simply an abbreviation for (Pi x in A)B when B does not contain x free. Similarly, A x B abbreviates (Sigma x in A)B when B does not depend on x.
