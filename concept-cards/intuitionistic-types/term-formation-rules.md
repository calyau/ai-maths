# Quick Definition

The term formation rules (Section 2.3) specify how terms of each type are constructed, organized as introduction and elimination rules in the style of Gentzen's natural deduction.

# Core Definition

Martin-Lof writes in Section 2.3: "Each rule of term formation will be classified a la Gentzen 1934 as an introduction or elimination rule associated with one of the basic types or type forming operations."

The rules are:

- **Variables** (2.3.1) and **Constants** (2.3.2): base cases for terms.
- **Pi-introduction** (2.3.3): lambda-abstraction.
- **Pi-elimination** (2.3.4): application.
- **Sigma-introduction** (2.3.5): pairing.
- **Sigma-elimination** (2.3.6): the E operator.
- **+-introduction** (2.3.7): injection (i and j).
- **+-elimination** (2.3.8): definition by cases (D operator).
- **N_n-introduction** (2.3.9) and **N_n-elimination** (2.3.10): finite type constants and R_n.
- **N-introduction** (2.3.11): 0 and successor s.
- **N-elimination** (2.3.12): primitive recursion (R operator).
- **V-introduction** (2.3.13): the reflection principle.
- **Type conversion** (2.3.14): a structural rule, neither introduction nor elimination.

# Prerequisites

- type: Terms are always terms of some type.
- variable: Variables are the most basic terms (rule 2.3.1).
- type-formation-rules: The types must be formed before their terms can be specified.

# Key Properties

1. Introduction rules build canonical terms of a type (constructors).
2. Elimination rules use a term of a type to produce terms of other types (destructors/recursors).
3. Each type former (Pi, Sigma, +, N_n, N, V) has its own introduction and elimination rules, except V which has only introduction.
4. Type conversion (2.3.14) is a structural rule that allows changing the type of a term when the old and new types are convertible.
5. The rules follow Gentzen's natural deduction pattern, represented by inference schemata with premises above and conclusion below the line.

# Construction / Recognition

## Introduction vs. Elimination:
- **Introduction**: Produces a term of the type in question from components. Example: (lambda x)b[x] introduces a term of Pi type.
- **Elimination**: Consumes a term of the type in question to produce a term of another type. Example: b(a) eliminates a Pi type to get a term of B[a].

## Structural Rule:
- Type conversion is neither introduction nor elimination; it changes the type annotation without changing the term.

# Context & Application

The Gentzen-style organization of term formation into introduction/elimination pairs is fundamental. Each pair is harmonious: the elimination rule extracts exactly the information that the introduction rule put in. This is formalized by the contraction rules (Section 2.4), where applying an elimination to an introduced term reduces to a simpler expression. For example, (lambda x)b[x](a) contracts to b[a].

# Examples

The inference schemata from the text:

Pi-introduction:
```
  x in A
  b[x] in B[x]
  ─────────────────────────────
  (lambda x)b[x] in (Pi x in A)B[x]
```

Pi-elimination:
```
  b in (Pi x in A)B[x]    a in A
  ─────────────────────────────────
  b(a) in B[a]
```

# Common Errors

- **Error**: Treating type conversion as an introduction or elimination rule.
  **Correction**: Type conversion is explicitly classified as a structural rule, distinct from both introduction and elimination.

# Common Confusions

- **Confusion**: Thinking elimination rules "destroy" terms.
  **Clarification**: Elimination rules extract information from a term; the original term is not altered or consumed in the derivation. They produce new terms from existing ones.
