# Quick Definition

Two expressions a and b are convertible (a conv b) if there exists an expression c such that both a red c and b red c. Conversion is the equivalence relation generated by reduction; it captures definitional equality.

# Core Definition

Martin-Lof writes (2.4.2): "an expression a is said to convert into an expression b, abbreviated a conv b, if there is an expression c such that both a red c and b red c."

Conversion is proved to be an equivalence relation (2.4.4):
- **Reflexive**: a conv a (take c = a).
- **Symmetric**: if a conv b then b conv a (by symmetry of the definition).
- **Transitive**: if a conv b and b conv c, then a conv c (using the Church-Rosser property).

Section 2.4.6 identifies conversion with **definitional equality**: two types A and B are definitionally equal if A conv B; a term a of type A is definitionally equal to a term b of type B if both a conv b and A conv B.

# Prerequisites

- reduction: Conversion is defined in terms of reduction.

# Key Properties

1. Conversion is an equivalence relation (Corollary 2.4.4).
2. Two expressions in normal form are convertible if and only if they are syntactically identical (up to bound variable renaming).
3. The rule of type conversion (2.3.14) uses this relation: if a is a term of type A and A conv B, then a is also a term of type B.
4. Successive applications of type conversion can be condensed into one (consequence of transitivity).
5. Definitionally equal types denote the same abstract type; definitionally equal terms denote the same abstract object (2.4.6).

# Construction / Recognition

## To Verify Conversion:
Show that both expressions reduce to a common expression. In practice, reduce both to normal form and check identity.

## To Use:
The type conversion rule allows re-typing a term when its type converts to another type. This is essential for the well-typing of many constructions.

# Context & Application

Conversion is the central notion of computational equality in the theory. It is what allows the type system to recognize that, for example, 2 + 3 and 5 are the same natural number, even though they are syntactically different. The rule of type conversion (2.3.14) is the structural rule that propagates this equality through the type system.

Martin-Lof notes (2.4.6) that definitional equality "is a relation between linguistic expressions and not between the abstract entities which they denote (and which are the same). This explains why the rule of type conversion, unlike all the other rules of term formation, has no counterpart on the informal level."

# Examples

- (lambda x)x(a) conv a, because (lambda x)x(a) red a and a red a.
- R(0, d, (lambda x)(lambda y)e[x,y]) conv d, because the recursion base contraction gives R(0, d, ...) red d.
- If A conv B, then any term of type A is also a term of type B by the type conversion rule.

- The type conversion rule (2.3.14) uses conversion to allow retyping.

## Contrasts With
- contraction: Contraction is a single directed step; conversion is a symmetric equivalence.
- reduction: Reduction is directed (not symmetric); conversion is symmetric.

# Common Errors

- **Error**: Thinking conversion means one expression reduces to the other.
  **Correction**: Conversion means both reduce to a common third expression. Neither need reduce directly to the other.

# Common Confusions

- **Confusion**: Definitional equality vs. propositional equality.
  **Clarification**: Definitional equality (conversion) is a meta-level judgment about expressions. Propositional equality (the equality type E(a,b) defined in 3.2.2.1.6) is itself a type within the theory.
