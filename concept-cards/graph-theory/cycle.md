# Quick Definition
A cycle is formed by taking a path P = x_0 ... x_{k-1} with k >= 3 and adding the edge x_{k-1}x_0. Its length is k (the number of edges, which equals the number of vertices).

# Core Definition
If P = x_0 ... x_{k-1} is a path and k >= 3, then the graph C := P + x_{k-1}x_0 is called a cycle. The length of a cycle is its number of edges (or vertices); the cycle of length k is called a k-cycle and denoted by C^k (Diestel, p. 8).

# Prerequisites
- **Graph** — a cycle is a graph
- **Path** — a cycle is constructed from a path

# Key Properties
1. A cycle of length k has exactly k vertices and k edges
2. Every vertex in a cycle has degree 2
3. The minimum cycle length is 3 (a triangle = C^3)
4. A cycle is 2-connected

# Construction / Recognition
## To Recognize a Cycle
1. Verify the graph is connected
2. Verify every vertex has degree exactly 2
3. The number of vertices equals the number of edges

# Context & Application
Cycles are fundamental structures. Their absence characterizes trees and forests. Odd cycles characterize non-bipartite graphs.

# Relationships
## Builds Upon
- **path** — a cycle extends a path by closing it

## Enables
- **girth** — minimum cycle length
- **circumference** — maximum cycle length
- **bipartite-graph** — characterized by absence of odd cycles
- **cycle-space** — generated by cycles

## Contrasts With
- **path** — a path has two distinct ends; a cycle has none
- **euler-tour** — traverses all edges, not a subgraph structure

# Common Confusions
- **Confusion**: Confusing a cycle (a subgraph) with a closed walk
  **Clarification**: A cycle has all distinct vertices; a closed walk may repeat vertices
