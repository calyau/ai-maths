# Quick Definition

Translation t_a by a vector a is the isometry defined by t_a(x) = x + a. Translations are isometries but not linear operators (unless a = 0). They form a normal subgroup of the isometry group.

# Core Definition

Translation t_a by a vector a is the map defined by t_a(x) = x + a (Artin, p. 164). Translations are isometries because |t_a(u) - t_a(v)| = |(u+a) - (v+a)| = |u - v|. They are not linear operators because they don't send 0 to 0, except for the zero translation. The translations form the kernel of the homomorphism pi: M_n -> O_n, and thus a normal subgroup of M_n (Proposition 6.2.10).

# Prerequisites

- **Isometry** — Translations are a type of isometry

# Key Properties

1. t_a(x) = x + a
2. t_a * t_b = t_{a+b} (composition law)
3. t_a^{-1} = t_{-a}
4. phi * t_a = t_{phi(a)} * phi for orthogonal phi
5. Translations form a normal subgroup T of M_n
6. T is isomorphic to the additive group of R^n

# Construction / Recognition

## To Recognize:
1. An isometry with no fixed point that preserves orientation
2. An isometry whose linear part is the identity

# Context & Application

Translations are the simplest isometries. They form the kernel of the homomorphism from M_n to O_n, making them the "invisible" part of an isometry when viewed through the lens of the orthogonal group. The translation subgroup of a discrete group of isometries is crucial for classifying crystallographic groups.

# Examples

**Example 1** (p. 163): The symmetry group of a frieze pattern (6.1.3) is generated by a single translation.

**Example 2** (p. 166): Composition rules: t_a * t_b = t_{a+b} and phi * t_a = t_{phi(a)} * phi.

# Relationships

## Enables
- **Lattice (geometric)** — Translation groups of crystallographic groups
- **Frieze group** — Generated by a single translation and possibly reflections

## Contrasts With
- **Rotation (plane)** — Rotations fix a point; translations do not
- **Reflection (isometry)** — Reflections fix a line; translations do not

# Common Errors

- **Error**: Treating translations as linear operators
  **Correction**: Translations (with nonzero vector) don't fix the origin and so are not linear

# Common Confusions

- **Confusion**: Thinking translation vectors change when coordinates change by translation
  **Clarification**: When the origin shifts by t_w, translation vectors are unchanged (p. 171)
