# Quick Definition

A module can be described by choosing generators and specifying the relations among them. This data is encoded in a presentation matrix A, so that the module is isomorphic to R^m / AR^n.

# Core Definition

Left multiplication by an m x n R-matrix A defines a homomorphism R^n -> R^m. The quotient module V = R^m / AR^n is presented by the matrix A (Artin, p. 437). The m rows correspond to generators v_1, ..., v_m and the n columns correspond to relations. A relation vector Y in R^m satisfies BY = v_1 y_1 + ... + v_m y_m = 0. The columns of A form a complete set of relations (Proposition 14.5.1).

# Prerequisites

- **Finitely generated module** -- Must have a finite generating set
- **Free module** -- Presentation involves quotients of free modules

# Key Properties

1. V is generated by the images (v_1, ..., v_m) of the standard basis of R^m (14.5.1(a))
2. An element BY of V is zero iff Y = AX for some X (14.5.1(b))
3. The presentation matrix depends on the choice of generators and relations (14.5.6)
4. Operations that don't change the isomorphism class: left/right multiply by invertible matrices, delete zero columns, delete row-column pairs with 1 entry (14.5.7)

# Construction / Recognition

## To Construct:
1. Choose generators (v_1, ..., v_m) for V -- this gives a surjection R^m -> V
2. Find generators for the module of relations W (kernel of the surjection)
3. Assemble the relation vectors as columns of an m x n matrix A

# Context & Application

Presentation matrices are the main tool for describing modules concretely. Combined with Smith normal form, they yield the structure theorem for finitely generated modules over PIDs.

# Examples

**Example 1** (p. 437): The abelian group with generators v_1, v_2, v_3 and relations 3v_1 + 2v_2 + v_3 = 0, 8v_1 + 4v_2 + 2v_3 = 0, etc., is presented by the 3x4 matrix (14.5.4). After reduction, it presents Z/4Z.

**Example 2** (p. 437): The cyclic group C_5 is presented by the 1x1 matrix [5], since C_5 is isomorphic to Z/5Z.

# Relationships

## Builds Upon
- **Finitely generated module** -- Only finitely generated modules have finite presentations
- **Free module** -- Modules are presented as quotients of free modules

## Enables
- **Smith normal form** -- Simplifies the presentation matrix
- **Structure theorem for finitely generated modules over a PID** -- Proven via diagonalization of presentation matrices

# Common Errors

- **Error**: Confusing the number of generators/relations with an intrinsic property of the module
  **Correction**: The numbers m and n depend on choices and are not uniquely determined by the module (p. 441).
